# 需求测试

执行测试验证，确保需求实现符合预期。

## 命令格式

```
/req test <REQ-XXX>
```

---

## 执行流程

### 0. 解析需求路径

```bash
# 检查当前仓库绑定的项目
PROJECT=$(cat .claude/settings.local.json 2>/dev/null | jq -r '.requirementProject // empty')

if [ -n "$PROJECT" ]; then
    REQ_ROOT=~/.claude-requirements/projects/$PROJECT
else
    REQ_ROOT=docs/requirements
fi

REQ_ACTIVE=$REQ_ROOT/active
```

### 1. 前置检查

```python
if 状态 not in ["开发中", "测试中"]:
    print("❌ 错误：需求尚未完成开发")
    print("💡 请先执行：/req dev REQ-XXX")
    exit()

# 检查功能点完成情况
if 有未完成的功能点:
    print("⚠️ 警告：存在未完成的功能点")
    显示未完成列表
    print("是否继续进入测试？(y/n)")
```

### 2. 更新状态为「测试中」

- 修改元信息状态为「测试中」
- 勾选生命周期「测试中」

### 3. 加载测试要点

从需求文档提取测试要点：

```

- [ ] 部门创建时关联渠道
- [ ] 部门更新时修改渠道关联
- [ ] 获取可选渠道接口

业务规则测试：
- [ ] 上级部门未设置渠道，下级可任意选择
- [ ] 上级部门已设置渠道，下级必须设置且为子集
```

---

## 测试模式说明

### 全量测试
执行所有测试要点。

### 增量测试
仅测试未通过的项：`/req test REQ-XXX --failed`

### 强制通过
跳过失败项：`/req test REQ-XXX --force`（不推荐）

# v2.5.0

> 发布日期：2026-04-15
> 版本范围：v2.4.0..HEAD

## 新功能

- `/req:review-pr fetch-comments` 新子命令：拉取 PR 评论（Issue Comments + Review Comments），自动过滤 AI 自提交和本人评论，AI 逐条生成修改方案，用户确认后应用到代码
- `/req:new --from-issue=#N` 支持从 GitHub/Gitea issue 创建需求，issue 标题/正文作为多轮讨论的起点
- `/req:new-quick --from-issue=#N` 同上，快速修复流程也支持从 issue 导入
- `/req:do --from-issue=#N` 从 issue 读取描述后进入意图分析（不创建文档）
- `/req:done` 完成归档后检测需求元信息的 `issue` 字段，交互询问是否关闭关联 issue（gitea PATCH API / `gh issue close`）

## 优化

- 需求模板（REQ + QUICK）元信息新增 `issue` 字段，用于跨阶段关联 Git 平台 issue
- `_common.md`、`CLAUDE.md` 同步记录新字段和新子命令/选项

## 兼容性

- 旧需求文档无 `issue` 字段时视为 `-`，不影响已有命令；`validate-requirement.sh` 只校验章节头，不校验字段行，无需变更

# v2.4.0

> 发布日期：2026-04-15
> 版本范围：v2.3.2..HEAD

## 新功能

- `/req:review-pr` 审查 PR 时对比需求文档与实际实现，列出文档未记录的接口、文件清单差异、未覆盖的功能点
- `/req:commit` 在保护分支上检测到改动时，自动根据改动推断分支名（`feat/` 或 `fix/` 前缀 + 英文 slug）并切换
- REQ 需求模板新增「七、图示（可选）」章节，提供流程图 / 时序图 / 状态图三个 Mermaid 示例，统一 `neutral` 主题
- PRD 模板 2.2 用户旅程、3.2.1 功能流程升级为 Mermaid 图示（`journey` / `flowchart`）

## 优化

- 命令模型策略改为"仅在需要强制降级时显式指定 haiku"，其他继承用户默认模型，避免在未开 Opus 的账号上报错或违背用户省钱偏好
- `/req:pr`、`/req:done` 执行流程精简，降低交互冗余
- `/req:done` 降级到 haiku 模型，归档场景省 token

## 重构

- 需求文档章节重编号：新增「七、图示」后，原 七~十 顺延为 八~十一，`10.x` 子节改为 `11.x`；同步更新 `validate-requirement.sh`、所有命令文件、`dev-guide` / `requirement-analyzer` 两个 skill 的引用
- `validate-requirement.sh` 顺手修正与模板不一致的章节名（`五、接口需求` → `五、数据与交互`）

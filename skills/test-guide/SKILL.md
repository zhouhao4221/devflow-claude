# 测试引导助手

引导完成需求的功能测试验证。

## 触发场景

- 执行 `/req test` 命令进入测试模式
- 用户说"测试一下这个功能"
- 用户说"验证一下接口"
- 开发完成后需要验证

## 测试框架

### 测试分类

| 类型 | 说明 | 执行方式 |
|-----|------|---------|
| API 测试 | 接口功能验证 | 自动（api-tester） |
| 业务规则测试 | 业务逻辑验证 | 交互式 |
| 权限测试 | 权限控制验证 | 自动+交互式 |
| 边界测试 | 边界条件验证 | 交互式 |
| 性能测试 | 性能指标验证 | 可选 |

### 测试流程

```
1. 测试准备
   ├── 确认服务已启动
   ├── 准备测试数据
   └── 加载测试要点

2. 自动测试
   ├── API 接口测试
   ├── 参数校验测试
   └── 错误码验证

3. 交互式测试
   ├── 业务规则验证
   ├── 边界条件验证
   └── 特殊场景验证

4. 回归测试
   ├── 关联功能验证
   └── 兼容性验证

5. 测试报告
   └── 生成测试结果
```

## 测试数据管理

```

本次测试创建了以下数据：
- 部门：3 条

是否清理测试数据？(y/n)
```

## 最佳实践

1. **先自动后手动**：优先执行自动测试
2. **关注边界条件**：特别测试边界情况
3. **验证错误处理**：确保错误码正确
4. **检查权限控制**：验证权限配置生效
5. **记录测试过程**：便于问题追溯

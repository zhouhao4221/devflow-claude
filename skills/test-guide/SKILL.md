# 测试引导助手

仅在 `/req:test` 命令执行时激活，引导完成测试验证。

## 测试分类

| 类型 | 说明 | 执行方式 |
|-----|------|---------|
| API 测试 | 接口功能验证 | 自动 |
| 业务规则测试 | 业务逻辑验证 | 交互式 |
| 权限测试 | 权限控制验证 | 交互式 |
| 边界测试 | 边界条件验证 | 交互式 |

## 测试流程

```
1. 准备 → 确认服务启动、加载测试要点
2. API测试 → 接口、参数校验、错误码
3. 交互测试 → 业务规则、边界、特殊场景
4. 回归测试 → 关联功能验证
5. 报告 → 生成测试结果
```

## 测试结果处理

- **通过** → 继续下一项
- **失败** → 记录问题，建议返回开发
- **跳过** → 标记原因

## 测试报告内容

- 测试统计（通过/失败/跳过）
- 详情列表
- 失败项描述
- 后续建议

## 最佳实践

1. 先自动后手动
2. 关注边界条件
3. 验证错误处理
4. 检查权限控制
5. 记录测试过程

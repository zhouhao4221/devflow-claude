# 需求分析助手

仅在创建或编辑需求时激活，引导完善需求文档各章节。

## 分析框架

### 1. 理解背景
- 业务目标：解决什么问题？
- 用户价值：给谁用？
- 使用场景：何时使用？

### 2. 功能拆解
- 拆分为原子功能点
- 每个功能点可独立开发/测试
- 标注依赖关系

### 3. 业务规则
| 类型 | 内容 |
|------|------|
| 数据校验 | 必填、格式、长度 |
| 状态转换 | 流转规则 |
| 权限控制 | 操作权限 |
| 边界条件 | 特殊情况 |

### 4. 数据模型
- 新增/修改实体
- 表结构设计
- 字段说明

### 5. API 设计
- RESTful 风格
- 请求/响应参数
- 错误码

### 6. 文件改动
按分层评估：Model → Store → Biz → Controller → Router

### 7. 测试要点
- 正常流程
- 边界条件
- 异常情况
- 权限测试

## 输出要求

使用项目需求模板格式，内容完整可执行。
